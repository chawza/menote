# AGENTS.md

Guidelines for AI agents working on the MeNote project.

## Project Overview

MeNote is a markdown note-taking desktop app. Multi-user on one device, local-first, built with Tauri 2 + SvelteKit 5 + SQLite. Authentication will be password-only (no username/email). Users switch by clicking profile icon.

## Tech Stack

- **Frontend**: SvelteKit 2.x + Svelte 5 + TypeScript + Vite
- **Desktop**: Tauri 2.x (Rust backend)
- **Database**: SQLite via Diesel ORM with embedded migrations
- **Type Bridge**: tauri-specta + specta for TypeScript type generation from Rust types (NOT ts-rs — ts-rs is an unused dependency in Cargo.toml)
- **Component Viewer**: Storybook 10 + @storybook/sveltekit + @storybook/addon-svelte-csf
- **Package Manager**: Yarn

## Architecture

- Frontend communicates with Rust backend via Tauri's `invoke` API, wrapped through `src/lib/utils/tauri.ts` → `tryCommand()`
- All Tauri commands return `Result<T, AppError>` which tauri-specta wraps as `{ status: "ok", data: T } | { status: "error", error: AppError }`
- SQLite database is stored in the Tauri app data directory (`menote.sqlite`), created on first launch
- Migrations are embedded in the binary and run automatically on startup
- TypeScript bindings are auto-generated in debug builds and written to `src/bindings.ts`
- Frontend uses Svelte 5 runes (`$props()`, `$state()`, `$effect()`, `$derived()`) and snippet blocks
- Rust backend is modularized: `commands/` (Tauri command handlers), `models/` (Diesel models + DTOs), `error/` (AppError), `db.rs` (connection + migrations)

## Code Conventions

### Rust

- Tauri commands are annotated with `#[specta::specta]` and `#[tauri::command]`
- Commands are registered in `collect_commands![]` in `lib.rs` `run()` function
- Commands are organized into modules under `src-tauri/src/commands/` (e.g., `users.rs`). Add a new module file and register it in `commands/mod.rs`.
- Models and DTOs are combined per entity in `src-tauri/src/models/` (e.g., `notes.rs` has `Note` model + `NewNote`/`NoteDetail`/`UpdateNote` DTOs). Add a new module file and register it in `models/mod.rs`.
- DB connection and migration setup extracted to `src-tauri/src/db.rs` — use `establish_connection()` to get a connection, `setup_data_base()` in app setup.
- Error handling via `AppError` enum (in `src-tauri/src/error/mod.rs`) with helper constructors (`note_not_found()`, `missing_id()`, `validation()`, etc.) and `From` impls for automatic conversion
- Convert between Model → DTO manually in command handlers
- Follow `snake_case` naming for everything

### TypeScript / Svelte

- Strict TypeScript — no `any` types
- Svelte 5 syntax: `$props()`, `$state()`, `$effect()`, `$derived()`, `Snippet` type for slots
- Always use `tryCommand()` from `src/lib/utils/tauri.ts` to call backend commands — it handles errors and shows toast notifications
- Import types from `src/bindings.ts` (auto-generated)
- Import commands from `src/lib/utils/tauri.ts` which re-exports the `commands` object
- kebab-case for file names, PascalCase for component names
- Component files go in `src/lib/components/`
- **Every component must have a corresponding `*.stories.svelte` file** in the same directory. Create it when creating a new component. Use `defineMeta` from `@storybook/addon-svelte-csf` with `tags: ['autodocs']`. Cover key variants (e.g., open/closed states, different types).
- Stores go in `src/lib/stores/`
- Types go in `src/lib/types/`

### Type Generation

- Types are generated by tauri-specta, NOT ts-rs. The `ts-rs` crate in Cargo.toml is unused and should be removed.
- In debug mode (`yarn tauri dev`), bindings are auto-exported to `src/bindings.ts`
- Never manually edit `src/bindings.ts`
- To add types exposed to frontend: add `#[derive(Serialize, Deserialize, Type)]` on Rust structs with `specta::Type`
- Both the Rust struct and the TypeScript type will stay in sync automatically

### Database / Migrations

- Use Diesel migrations for ALL schema changes. Never modify schema directly.
- Migrations are in `src-tauri/migrations/` with `up.sql` and `down.sql`
- `schema.rs` is auto-generated by Diesel. Never edit it manually.
- Migrations run automatically on app startup via `embed_migrations!`
- Current schema issues (see PLAN.md Phase 1): missing foreign key on `notes.user_id`, `content` allows NULL, no CHECK constraint

## Adding New Features

### Adding a Tauri command

1. Define the Rust function in the appropriate module under `src-tauri/src/commands/` (or `lib.rs` for notes) with `#[specta::specta]` and `#[tauri::command]`
2. Register the module in `commands/mod.rs` if it's new
3. Add the command to `collect_commands![]` in `lib.rs` `run()` function
4. If using new types, define models/DTOs in `src-tauri/src/models/<entity>.rs` with `#[derive(Serialize, Deserialize, Type)]`
5. Register new model module in `models/mod.rs` if it's new
6. Run `yarn tauri dev` once to regenerate `src/bindings.ts`
7. Call from frontend: `tryCommand(() => commands.my_new_command(args))`

### Adding a database model

1. Create migration: `cd src-tauri && diesel migration generate migration_name`
2. Write `up.sql` and `down.sql` in the migration folder
3. Run `diesel migration run` to apply and regenerate `schema.rs`
4. Add model struct + DTOs to `src-tauri/src/models/<entity>.rs` with Diesel derive macros
5. Register in `models/mod.rs`
6. Add command(s) in `src-tauri/src/commands/<entity>.rs` or `lib.rs`

### Creating frontend routes

1. Add route in `src/routes/` following SvelteKit file-based routing
2. Create `+page.svelte` for the route component
3. SSR is disabled (`ssr = false` in `+layout.ts`) — use `+page.ts` not `+page.server.ts`

## Common Patterns

### Calling a backend command from frontend

```typescript
import { tryCommand, commands } from "$lib/utils/tauri";

const notes = await tryCommand(() => commands.getNotes(userId));
```

### Defining a new Tauri command

```rust
#[specta::specta]
#[tauri::command]
fn my_command(arg: i32) -> Result<MyDto, AppError> {
    use crate::schema::my_table::dsl::*;
    let conn = &mut establish_connection();
    // query with Diesel...
    Ok(my_dto)
}
```

### Model → DTO conversion

```rust
let dto = MyDto {
    id: model.id.ok_or_else(|| AppError::Internal("Missing id".into()))?,
    // ...other fields
};
```
